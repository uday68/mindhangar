## Translation Quality Guidelines

### 1. Accuracy
- Translations must be accurate and contextually appropriate
- Technical terms should be translated consistently
- UI labels should be concise and clear

### 2. Cultural Sensitivity
- Use respectful and inclusive language
- Avoid regional biases
- Consider religious and cultural diversity

### 3. Consistency
- Use same translations for repeated terms
- Maintain consistent tone across all strings
- Follow platform-specific conventions

### 4. Readability
- Use simple, everyday language
- Avoid overly formal or technical language
- Ensure translations fit UI constraints

## Technical Implementation

### How It Works

```typescript
// 1. User selects language
setLanguage('hi'); // Hindi

// 2. I18nContext updates
const { t, currentLanguage } = useI18n();

// 3. Components use t() function
<h1>{t('dashboard.welcome', { name: 'Rahul' })}</h1>
// Output: "वापस आपका स्वागत है, Rahul!"

// 4. All UI updates automatically
```

### File Structure
```
src/i18n/
├── index.ts (I18n setup)
├── messages/
```
